第18章 PSE架构：五层引擎与心智六论

学习目标：完成本章后你应能够：(1)描述PSE五层架构的职责；(2)简述心智六论各自解决的核心问题；(3)理解PSE-I理论注入如何将UPLS语法转化为可计算模型；(4)区分PSE四个仿真阶段的功能定位；(5)了解PSE的三个输入通道及UNIS的神经似然输入。

⚠️ 实证成熟度声明：PSE当前处于概念设计阶段（v0.1-alpha）。本章中的所有数值、仿真结果和预测均为基于理论推导的模拟演示，不可用于临床决策。

18.1 五层架构：架构、算法、参数、方法论与协议

心理系统工程引擎(PSE) 是INTRE框架的计算核心。它将功能层的静态描述语法(M-R-D-S)和操作语法(G-M-A-R)转化为可运行、可校准、可预测的动力学仿真系统。PSE采用五层架构，每一层承担不同的工程职责。

第一层：架构层(Architecture Layer)——PSE的静态本体论基础。包含： - M-R-D-S内感架构：42维状态空间的定义 - G-M-A-R操作语法：四个原子操作的定义 - S-P外感架构：环境输入的六层来源分类 - N-S-R元架构：最高阶组织原则，在MDT优化目标函数中体现为长期功能完整性指数\(O_i(T)\)的驱动

架构层变化最慢，只在INTRE框架版本升级时修改。

第二层：算法层(Algorithm Layer)——PSE的“物理引擎”，由心智六论构成（详见18.2节）。算法层只包含函数形式、拓扑关系和约束规则，不包含具体参数值。

第三层：参数层(Parameter Layer)——为算法层的方程提供具体的概率分布（先验），编码既有知识的不确定性。由五个标准知识库构成： - SPMC-L：PCUI映射的参数分布 - SPER-L：S-P典型模式与生态位的参数配置 - SPSM-L：因果路径模板、动力学参数先验 - SPDC-L：G-M-A-R操作的执行成本、预期收益的先验 - SPOT-L：抽象操作转译为具体情景化任务的规则库

参数层变化最快，每次新个体数据进入PSE-C校准都会更新。

第四层：方法论层(Methodology Layer)——规定PSE如何被实际运行，包含四个阶段： - PSE-I（理论注入）：将算法层、架构层、参数层编译为可执行仿真代码 - PSE-M（大规模蒙特卡罗仿真）：在参数空间中采样，运行数百万虚拟智能体的状态演化 - PSE-P（先验参数提取）：从PSE-M输出中为每个生态位提取参数分布 - PSE-C（真实世界贝叶斯校准）：将个体观测数据注入，通过贝叶斯更新实现个体化状态估计

第五层：协议层(Protocol Layer)——PSE与临床实践之间的接口，将计算输出转化为标准化工作流程：

协议 名称 功能
VTS-P 变量-结构协议 临床评估数据 → M-R-D-S状态向量
ETS-P 环境-结构协议 环境信息 → S-P影响向量
SID-P 系统集成诊断协议 状态向量+S-P向量 → 系统优化蓝图
MMD-P 多维度元调度协议 优化蓝图 → 五维调度指令
STO-P 结构-操作协议 抽象操作 → 情景化操作清单

PSE的输入通道：PSE是一个多模态贝叶斯融合引擎，其输入来自三个通道： - 行为通道：传统心理量表、行为任务（如Stroop、N-back）、生态瞬时评估(EMA)——由UBMS（统一行为测量系统）提供标准化观测函数、工具桥接和个体化贝叶斯校准，最终输出行为似然 p(y_b | Ψ_t) - 语义通道：通过UPLS的PSL将自然语言编译为PCUI，再映射为M-R-D-S状态向量的先验分布 - 神经通道：通过UNIS（统一神经接口系统）将原始神经信号（EEG、fNIRS、PPG等）转换为神经似然 \( p(y_n \mid \Psi_t) \)，以/func_likelihood协议传入PSE

PSE将这三个通道的似然进行贝叶斯融合，输出状态后验分布。同时，PSE通过/state_estimate和/calibration_feedback协议将状态估计和校准残差反馈给UNIS，实现神经模型的在线个体化微调。

层间依赖关系：架构层 → 算法层 → 参数层 → 方法论层 → 协议层（单向）。下层变化影响上层，但上层变化不应波及下层。这实现了关注点分离：测量工具改进只影响参数层，不会迫使架构层修改。

18.2 心智六论：MCT至MET的职责分工

心智六论是PSE算法层的核心，每个理论模块规定一类动力学规则。

MCT：心智编译理论(Mental Compilation Theory)——解决“输入问题”：将非结构化临床叙述编译为M-R-D-S状态向量。编译流程：词法分析（PSL切分）→ 概念映射（PCUI匹配）→ 语境消歧 → 状态向量填充 → 不确定性量化（输出概率分布\(p(\Psi_0)\)）。

MTT：心智拓扑理论(Mental Topology Theory)——解决“结构问题”：状态空间中各分量之间的依赖关系。将心理系统建模为有向加权图\(G=(V,E)\)，节点是M-R-D-S子单元，边是有向依赖关系。核心输出： - 依赖矩阵：42×42的权重矩阵 - 枢纽节点识别：计算出度、入度、介数中心性（M4通常最高） - 级联路径分析：识别典型传播路径 - 反馈环路检测：识别正/负反馈回路

MKT：心智动力理论(Mental Kinetics Theory)——解决“速率问题”：状态分量以什么速率、遵循什么函数形式变化。一般形式： \[ \frac{d\Psi_i}{dt} = f_i(\Psi, E, u; \theta) \] 区分三类动力学：资源动力学（消耗+恢复，恢复不对称定律）、调度动力学（阈值切换）、结构动力学（学习规则，慢变量）。

MDT：心智调度理论(Mental Dispatch Theory)——解决“干预规划问题”：在给定当前状态、目标状态和资源约束下，选择最优G-M-A-R操作序列。形式化为约束优化问题： \[ \max_{O} \; \mathbb{E}[\Delta O_i(T) \mid \Psi(t), E(t), \text{约束}] \] 考虑资源约束、前置条件约束、时序约束。输出优先级排序的操作序列及预期收益、成本、成功概率。

MPT：心智塑性理论(Mental Plasticity Theory)——解决“结构改变的动力学”：STR层在什么条件下、以什么速率发生持久变化。三个核心原则： - 塑性窗口原则：结构改变需要去稳定化→修改→巩固三阶段 - 能量-塑性耦合：结构改变深度受可用资源限制 - 巩固不稳定原则：新修改的结构在初始巩固期内不稳定

MET：心智生态理论(Mental Ecology Theory)——解决“环境-系统耦合问题”：S-P环境输入如何调制M-R-D-S状态演化。定义两种耦合方向： - 下行耦合（环境→系统）：S-P向量作为强制输入进入状态演化方程 - 上行耦合（系统→环境）：系统状态影响个体如何选择和修改环境（生态位建构）

MET还定义生态位(ecological niche)：个体典型S-P环境配置与M-R-D-S基线的联合分布，用于冷启动先验匹配。

18.2.1 定律速查：26条核心命题

说明：以下26条定律构成心智六论的形式化核心。完整推导及数学细节见01-X-03（心智六论形式化规范）。验证状态分为：F=已有实证支持；X=理论推导/待验证；P=预测性子单元/前瞻性实证待完成。

编号 定律名称 核心陈述 验证状态
MCT-1 编译保真界 现象层主诉到功能层状态向量的编译保真度不超过PSL词法分析×PCUI映射×语境消歧三阶段精度的乘积 X
MCT-2 不确定性单调性 编译输出的后验不确定性不低于前序阶段中最大不确定性 X
MCT-3 语境消歧优先 当多义词存在语境关键词匹配时，优先采用高置信度消歧规则，仅在置信度<0.5时退回人工确认 F
MCT-4 主映射收敛 同一构念经多源观测（行为/生理/自陈）独立编译后，M-R-D-S坐标的主映射应趋于收敛 P
MCT-5 次映射衰减 次映射对状态估计的贡献权重随层级距离指数衰减 X

编号 定律名称 核心陈述 验证状态
MTT-1 依赖有向性 M-R-D-S子单元之间的功能依赖关系构成有向无环图（DAG），不存在循环依赖 X
MTT-2 枢纽集中度 在42维依赖矩阵中，枢纽节点的出度+入度之和与介数中心性呈正相关，M4（情绪模块）通常具有最高枢纽度 P
MTT-3 级联深度约束 任意两个非相邻子单元之间的功能影响最多经3个枢纽节点即可完成传递（小世界特性） X
MTT-4 拓扑鲁棒性 依赖矩阵的全局结构在个体间高度保守（跨个体相关系数>0.85），局部权重可调 P

编号 定律名称 核心陈述 验证状态

MKT-2 调度阈值切换 DSP-F与DSP-B之间的控制权切换遵循滞后阈值函数，切换阈值为资源存量的非线性函数 X

MKT-4 耦合刚度分级 R→D耦合（资源影响调度）> D→S耦合（调度影响结构）> S→R耦合（结构影响资源） X
MKT-5 基线回归弹性 系统在扰动后向个体基线回归的速率与偏离幅度成正比（弹性系数因人而异） P

MDT-3 时序匹配 操作的效果时间尺度（GATE秒-时、MODULATE时-天、ALIGN天-月、RELINK月-年）须与问题的自然恢复周期匹配，否则干预效率<最优效率的50% X
MDT-4 边际递减 同一操作在同一靶点上的累积收益呈对数递减，连续应用超过3次后增量效果低于首次的20% P

MPT-4 塑性窗口时间锁 重大结构改变（RELINK）需匹配睡眠依赖的再巩固时间窗，脱离该时间窗的效率降低>50% F

### 18.3 理论注入：PSE-I与文化变体编译

PSE-I是将抽象理论规范转化为可运行计算机代码的编译阶段。

输入：架构层（42维状态空间定义、操作合法性矩阵）、算法层（依赖矩阵、微分方程形式、优化目标函数）、参数层（先验分布）。

编译过程转换为： 1. 状态空间声明（数据结构） 2. 动力学函数库（\(f_i\)的代码实现） 3. 依赖矩阵初始化 4. 参数采样器（用于PSE-M） 5. 观测函数（将状态映射到观测空间） 6. 协议模板（VTS-P等决策树骨架）

输出：可执行的PSE仿真环境（Docker容器或Python库），以及标准API供PSE-M/PSE-P/PSE-C调用。

文化变体的注入：UPLS的PSL-L中的文化变体参数在PSE-I阶段被注入，使仿真可针对不同文化背景调整。文化变体参数初始值基于跨文化文献（F级），跨文化等值性有待PSE-C验证（P级）。

验证与测试：PSE-I输出需通过单元测试、集成测试、回归测试。PSE开源引擎快速入门见附录C。

本章小结：PSE是INTRE框架的计算核心，采用五层架构将功能层静态描述转化为可运行的动力学仿真系统。架构层定义坐标系统，算法层提供演化规则（心智六论），参数层提供先验分布，方法论层规定四阶段仿真流程（I/M/P/C），协议层提供临床接口。PSE的输入来自三个通道：行为、语义（UPLS）和神经（UNIS），通过贝叶斯融合实现多模态状态估计。心智六论分工明确：MCT编译输入，MTT描述拓扑，MKT提供速率方程，MDT规划干预，MPT约束结构改变，MET耦合环境。PSE-I是将理论规范编译为可执行代码的注入阶段，文化变体在此纳入。PSE的设计类比于数值天气预报——它不是在已有数据上拟合统计规律，而是从理论方程出发，通过同化观测数据来逼近个体真实状态。

关键术语：PSE，五层架构，心智六论，MCT/MTT/MKT/MDT/MPT/MET，PSE-I/M/P/C，三通道输入，UNIS

常见误区： - ❌ “PSE是已经工作的引擎” → ✅ PSE当前处于概念蓝图阶段，大规模校准和实证验证未完成。 - ❌ “心智六论是六个独立的模型” → ✅ 它们是同一个动力学系统的不同侧面，通过状态向量\(\Psi(t)\)和依赖矩阵耦合。 - ❌ “PSE只需要行为数据” → ✅ PSE是多模态融合引擎，需要行为、语义（UPLS）和神经（UNIS）三个通道的输入。
