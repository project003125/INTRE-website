附录J：PCUI规范——角色标签、映射与消歧

定位：第14章（概念标识）的配套速查手册，供研究者和开发者日常查阅。

学习目标：能够独立完成一个心理概念的PCUI编码（含ModuleSignature和元数据标注）；能够快速查阅同义词映射表和一词多义消歧规则。

J.1 PCUI编码格式

PCUI（Psychological Concept Unique Identifier）为每个心理概念分配唯一的结构化编码。完整格式如下：

[CONCEPT_NAME].[DOMAIN].v[VERSION].[SEQUENCE]#[MODULE_SIGNATURE]
字段 说明 示例 必填
CONCEPT_NAME 概念规范英文名称（全大写下划线分隔） CORE_BELIEF_NEGATIVE 是
DOMAIN 固定为UPLS UPLS 是
VERSION 主版本号.次版本号 v3.0 是
SEQUENCE 同一版本内的顺序号（三位数字） 007 是
MODULE_SIGNATURE 模块角色签名（见J.2） #M2-STR-R 是

简化格式（日常使用、内部文档、快速标注）： CONCEPT_NAME#MODULE_SIGNATURE 示例：PROCRASTINATION#M3C+M2I+M6I

J.2 ModuleSignature角色标签速查表

ModuleSignature使用角色标签标注概念涉及哪些模块及其功能角色。标签组合格式：M1C+M2I+M3S（模块号+角色，多个用+连接）。

标签 名称 含义 用法示例 互斥规则
C Core 核心失调位点 M4C = M4是该概念的核心模块 每个PCUI至少1个C，通常不超过2个
I Impact 次级受冲击模块 M2I = M2受冲击 不能与C同时用于同一模块
S Support 支撑/资源供给模块 M1S = M1提供能量支撑 不能与C同时用于同一模块
T Trigger 触发条件模块 M4T = M4状态变化触发 可选
O Output 行为/体验输出通道 M6O = M6是行为输出通道 可选

完整示例：

PCUI编码 ModuleSignature 解读
CORE_BELIEF_NEGATIVE.UPLS.v3.0.007#M2C M2C 负性核心信念：M2认知模块为核心
CHRONIC_ANXIETY.UPLS.v2.0.001#M4C+M2I+M3I+M1S+M6O M4C+M2I+M3I+M1S+M6O 慢性焦虑：M4为核心；M2/M3受冲击；M1支撑；M6输出
PROCRASTINATION.UPLS.v3.0.102#M3C+M2I+M6I M3C+M2I+M6I 拖延：M3意志为核心；M2/M6受冲击

拖延的亚型细分（同一PCUI条目内）：

亚型 M-R-D-S指纹 ModuleSignature变体
资源枯竭型 M3-RES-D↓ M3C
规则扭曲型 M2-STR-R（完美主义） M2C+M6I
情绪驱动型 M4-STR-C（任务-威胁连接） M4C+M6I

J.4 一词多义消歧规则

同一术语在不同语境中可能映射到不同的PCUI条目。

术语 语境关键词/条件 PCUI映射 置信度 缺省行为
防御机制 “无意识”“焦虑”“压抑” DEFENSE_MECHANISM 高(≥0.8) —
“回避想法”“分心”“逃避思考” COGNITIVE_AVOIDANCE 中(0.6-0.7) —
无明确上下文 — <0.5 返回备选项，要求人工确认
抑郁 “情绪低落”“没劲”“快感缺失” DEPRESSED_MOOD 高 —
“PHQ-9≥10”或“诊断为MDD” DEPRESSION_SYMPTOMS 高 —
“抑郁型人格”“长期负性” DEPRESSIVE_PERSONALITY 中 —
焦虑 “担心未来”“灾难化思维” ANXIETY_COGNITIVE 高 —
“心慌、出汗、发抖” ANXIETY_SOMATIC 高 —
“回避行为”“不敢出门” ANXIETY_BEHAVIORAL 高 —

消歧流程：术语进入 → 提取语境关键词 → 匹配规则 → 置信度>0.7直接映射；0.5-0.7输出候选列表；<0.5要求人工确认。

J.5 PCUI条目元数据标注模板

一个完整的PCUI条目应包含以下元数据。带\*号为必填项。

元数据字段 类型 说明 示例 必填
规范名称 字符串 概念英文全大写，下划线分隔 PROCRASTINATION \*
PCUI编码 字符串 完整编码（含版本和模块签名） PROCRASTINATION.UPLS.v3.0.102#M3C+M2I+M6I \*
版本 语义版本号 vMAJOR.MINOR.PATCH v3.0.102 \*
创建日期 ISO日期 YYYY-MM-DD 2024-06-15 \*
最后修订 ISO日期 YYYY-MM-DD 2026-03-10 \*
核心定义 文本 功能层定义，≤300字 “拖延是指个体在明知延迟会导致更差结果的情况下，仍自愿延迟计划行动的行为模式。” \*
M-R-D-S指纹 列表 亚型 → M-R-D-S坐标 见J.2拖延亚型表 如有多亚型则必填
ModuleSignature 字符串 C/I/S/T/O标签组合 M3C+M2I+M6I \*
多流派解释 键值对 流派名 → 解释文本 CBT: “自动负性思维…”; ACT: “经验性回避…” 至少2个流派
G-M-A-R映射 列表 亚型 → 推荐操作序列 资源枯竭型：MODULATE+优先 \*
可信度 0-10整数 多专家共识评分 8 \*
IET等级 枚举 D/L/F/X/P/N/E/R F \*
边界注释 文本 不包括什么、争议点 “排除ADHD执行功能障碍导致的启动困难” \*
强反例条件 文本 触发修订的证据条件 “若发现第四种不可被现有指纹解释的拖延亚型，且在≥3个独立样本中复现，则条目需扩展” \*
异议档案 文本/链接 PCV-P反对意见记录 评审记录#PCV-P-2024-102 否
文化变体 键值对 文化 → 权重/基线调整 东亚：M1-DSP-B权重×1.3 否

J.6 快速编译检查清单

研究人员在创建新PCUI条目时，请逐项确认：

  1. 唯一性检查：是否检索了现有PCUI库避免重复？□
  2. 模块签名：ModuleSignature是否至少包含1个C标签？□
  3. 多流派覆盖：是否至少有2个流派的术语映射或解释？□
  4. 证据标注：可信度（0-10）和IET等级是否已标注？□
  5. 边界声明：边界注释是否明确排除了不适用范围？□
  6. 可证伪性：强反例触发条件是否可检验（含具体阈值或统计标准）？□
  7. 数值诚实：所有数值是否标注了“示意性示例”或提供了实证来源？□

通过标准：以上7项全部勾选，方可提交PCF-P编译流程。
