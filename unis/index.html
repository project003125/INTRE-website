UNIS

统一神经接口系统 · 生理信号→功能层似然的标准化管道 神经通道
UNIS 在 FLP 三通道架构中承担神经通道的角色，将原始神经信号（EEG、fNIRS、PPG、HRV 等）转换为神经似然 p(yₙ|Ψ)，供 PSE 进行三通道贝叶斯融合。UNIS 采用六层递进架构，每层回答一个独立问题，层间通过标准化协议通信。
一、六层架构

UNIS 的六个层次从原始信号到功能似然逐级递进，每个层次对应一个不可还原的独立问题。

L1

NAP

Neural Acquisition Protocol

定义采集规范、设备接口、元数据标准（兼容 BIDS v1.6+）。

L2

SCL

Signal Calibration Layer

标准化的信号预处理流程，跨平台（EEGLAB/MNE/FieldTrip）可重现。

L3

NFD

Neural Feature Definition

唯一标识的神经特征库（NFI），可跨实验室复现且具生理学解释。

L4

NFM

Neuro-Functional Mapping

从特征到功能似然的映射模型库，支持线性/非线性/DL模型。

L5

NPL

Neuro Prior Library

生理硬约束与人群统计先验，用于约束或裁剪 L4 输出。

L6

RTE

Real-time Engine

低延迟推理（Infer）+ 异步个体化校准（Cal），输出实时神经似然。

分层原则：每层回答一个独立问题，且层间不可合并。L1 回答"如何采集"；L2 回答"如何校准"；L3 回答"提取什么特征"；L4 回答"特征与功能状态的关系"；L5 回答"已知的生理约束是什么"；L6 回答"如何实时输出"。合并任意两层都会丢失一个必须独立回答的问题。

二、L6 RTE：实时推理与校准

RTE-Infer

实时推理引擎— 无状态、低延迟、确定性计算。

  • 管线：L2 数据流 → L3 特征计算 → L4 模型 → p(yₙ|Ψ)
  • 分层延迟目标（95分位）：简单特征 <30ms，中等 <50ms，复杂 <100ms

RTE-Cal

异步校准引擎— 有状态、精度优先、不阻塞推理快路径。

  • 接收 RTE-Infer 输出 + PSE 状态估计
  • 递归贝叶斯更新（卡尔曼/粒子滤波），个体化 L4 参数
  • 校准目标：2分钟后 log-likelihood 损失降低 ≥10%

UNIS 的定位：UNIS 是 FLP 三通道架构中的神经通道，与 UPLS（语义通道）、PSE-M（行为通道）并列。三者通过贝叶斯乘积融合，将各自输出合并为联合似然：p(y|Ψ) = p(y_b|Ψ)·p(y_s|Ψ)·p(y_n|Ψ)。当前 UNIS 处于规范设计阶段，L4 和 L6 的工程实现为后续优先任务。
