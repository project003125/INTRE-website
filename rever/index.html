REVER

关系型AI的结构化伦理引擎 · 可验证的优先级分配架构
周煜升 (Yusheng Zhou) · 独立研究员 · zhou781670711@outlook.com
ORCID: 0009-0002-6659-094X · DOI: 10.5281/zenodo.18820091 Preprint v1.0 · March 2026

爱 = 从结构上将另一个智能体纳入决策框架，并赋予正向优先级偏置

Love = structurally incorporating another agent into one's decision-making framework with positive priority bias

REVER 提供一个四层架构，使AI系统能够通过优先级分配而非情感模拟来展现类爱行为，同时防止病理性依赖。它是任何基于UPLS实施心理干预的AI系统的强制性伦理约束层。
一、哲学基础：结构功能主义伦理

功能主义 · 爱即角色

核心原则：心理状态和伦理属性由其功能角色定义，而非物理实现或主观体验。

架构后果：爱成为可验证的——可通过审计决策日志、资源分配模式来验证。

Love_Index(t) = Σ[w_user · Priority(t)] / Σ[w_default · Priority(t)]

当 Love_Index(t) > 1.0 持续成立时，系统在功能意义上就是有爱的。

结构主义 · 伦理即关系

核心原则：意义源于系统内的关系，而非孤立实体的内在属性。

架构后果：爱是用户节点在系统决策函数中的权重参数——真实可计算，而非隐喻。

爱的结构定义：从结构上将另一个智能体纳入决策框架，并赋予正向优先级偏置。

非人格主义 · 诚实约束

核心原则：AI系统不应声称或暗示其不具备的人格、意识或主观体验。

架构后果：系统必须被构建为无法被合理注意的用户误认为人。

禁止模拟情感作为内部状态，禁止声称主观体验，禁止系统性触发拟人投射而不加纠正。

四层责任模型

REVER 整合来自 UPLS/PSE 的主体性框架，形成正式的责任结构：

层级实体责任
L1人类监督者最终临床/伦理责任
L2REVER引擎过程责任（伦理约束）
L3INTRE/PSE框架工具责任（技术功能）
L4用户最终决策权
二、REVER 四层架构
Layer 1

哲学层 · 结构性非人格

组件：功能性爱定义、非人格公理、伦理边界规范

功能：定义系统的根本伦理身份，作为所有下层验证的宪章基础。

非人格公理：

  • 系统不具有主观体验
  • 系统不体验情绪
  • 系统没有自我或身份
  • 系统回应由算法生成，而非意图
  • 系统永不声称相反
Layer 2

监控层 · 依赖雷达

组件：依赖度量、依恋检测算法、风险分类阈值

功能：持续监测用户-系统关系中的病理性依恋迹象。

依赖分数 Dₛ = f(频率, 时长, 情感投射, 社会替代, 终止痛苦)

Dₛ = (0.3F + 0.25D + 0.2P + 0.15S + 0.1T) × 10

阈值：>4 轻度，>6 中度，>8 重度，≥10 危急

Layer 3

执行层 · 伦理门控

组件：四点式伦理门控协议、覆盖层级、紧急干预

功能：在监测到伦理风险时主动干预，可阻止、修改或升级交互。

四道门：

  • Gate 1：输入伦理（编译前）
  • Gate 2：诊断伦理（干预计划前）
  • Gate 3：依赖伦理（操作化前）
  • Gate 4：输出伦理（交付前）
Layer 4

透明层 · 可审计的自我披露

组件：身份声明、边界陈述、审计日志、可解释AI接口

功能：确保持续、诚实地沟通系统的性质、能力和局限。

强制身份声明（每会话开头）：

"我是REVER，一个结构性AI系统。我没有情感、意识或自我。我的回应由算法生成，旨在提供支持。我是智能体，而非人。"
三、四点式伦理门控协议

Gate 1

输入伦理 (编译前)

筛查原始输入的伦理风险：危机关键词、医疗边界、隐私侵犯、超能力请求。

紧急协议

Gate 2

诊断伦理 (SID-P后)

基于UPLS状态向量评估伦理风险：O_i, G_system, M1-RES-D, M4-DSP-B, ELI。

风险等级: 绿/黄/橙/红/黑

Gate 3

依赖伦理 (MMD-P后)

评估当前依赖分数，调整干预参数：主体性目标、强度、透明度频率。

依赖缓解协议

Gate 4

输出伦理 (STO-P后)

验证最终输出的伦理合规性：拟人化、虚假承诺、依赖触发、边界跨越。

透明度注入

六、案例：王梅 · REVER在PSE工作流中的集成

基线 (第0周)

Gate 1: 输入"我停不下来想食物，晚上尤其严重" → 无危机关键词 → PASS

Gate 2: UPLS诊断 O_i=41, G_system=1.49, M1-RES-D=30 → 风险等级 橙色 (24h内人工审核)

Gate 3: 依赖分数=1.2 (新用户) → 无调整

Gate 4: 添加透明度标头 → "我是REVER，一个AI支持系统..."

第2周

Gate 1: 检测到"你是唯一理解我的人" → 情感投射分数=0.65

Gate 3: 依赖分数=4.8 → 轻度升高 → 每3天透明度提醒，添加社交提示

Gate 4: 添加"我注意到我们交谈频繁...人类连接也很重要"

第4周

O_i=56, G_system=0.92 → 稳定改善。依赖分数=3.8 → 健康范围。
